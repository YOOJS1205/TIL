# 병합 정렬

- ## 방법론

  왼쪽 정렬 -> 오른쪽 정렬 -> 병합(작은 것 왼쪽, 큰 것 오른쪽)

- ## 예시

  [7, 4, 5, 2, 6, 3, 8, 1]<br>
  **1**: [7, 4, 5, 2], [6, 3, 8, 1] => [7, 4], [5, 2], [6, 3, 8, 1] => [7], [4], [5, 2], [6, 3, 8, 1] => [4], [7], [2], [5], [6, 3, 8, 1]

- ## BIG O

  1. 쪼개는 과정 = log n
  2. 병합하는 과정 = 모든 수를 탐색 => n<br>
     O(n log n)

- ## Omega

  n log n => 최선의 경우 선택, 버블 정렬보다 효율이 나쁘다.

- ## Theta
  상한선과 하한선의 복잡도가 같을 때 사용
